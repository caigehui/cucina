# daily-report 每日美股研报模块

一次性把四个模块的产出汇总成**一份可读、分层、客观**的《每日美股研报》,**保存为 Markdown 和 PDF 两个版本**到项目文件夹(供用户随时打开)。始终简体中文。

日报的内容仍是宏观、事件、社媒、科技链和持仓提示,不是单只股票的六维技术分析;但视觉风格、Markdown 结构和 PDF 呈现要对齐 `technical-analysis`:深色科技风、结论先行、自然标题、数据 HUD 感、页眉页脚清晰,不再使用机械编号标题。

## 运行时机与定位
- 默认按用户本机美西时区 `America/Los_Angeles` 在美股常规交易日运行两次:
  - **美西 04:30**:常规开盘美西 06:30 / 美东 09:30 前 2 小时,等价于美东 07:30,输出**「隔夜复盘 + 当日盘前展望」**研报。
  - **美西 15:00**:常规收盘美西 13:00 / 美东 16:00 后 2 小时,等价于美东 18:00,输出**「当日收盘复盘 + 盘后跟踪」**研报。
- 盘前版回顾上一交易日美股收盘、扫描隔夜要闻,预览当日交易和未来事件日历;盘后版复盘当日收盘、盘后新闻/财报/指引、盘后异动和次日待验证事项。
- 周末/美股休市或无新交易日时,自动转为**「休市复盘 + 下个交易日展望」**轻量版。

## 生成流程(顺序执行)

1. **定时间**:bash 取 `TZ='America/Los_Angeles' date`、`TZ='America/New_York' date` 与 `TZ='Asia/Shanghai' date`,确定上一交易日与当日。
2. **宏观与事件**:按 `macro-events` 流程采集货币政策/通胀就业/财政债务/贸易政策/地缘新闻 + 未来 1-2 周事件日历(FOMC、CPI/PCE、NFP、四巫日、国债拍卖、重点财报)。给已定价 vs 未定价判断。
3. **社媒情绪(重点模块)**:按 `wsb-sentiment` 流程做 **WSB-first 社媒雷达**。必须列出 WSB 散户最关注的个股/ETF/加密资产、喊单方向、常见玩法、证据摘要、HYPE 阶段、风险/反向信号;X/Twitter 只做交叉验证和叙事扩散补充,不能替代 WSB。若社媒源不可得,必须明确数据缺口,不得凭空补造热门票。
4. **科技股/赛道**:按 `tech-research` 方法论,挑当日**最值得讲的 1-3 个科技/AI 供应链看点**(财报、卡点验证信号、出口管制传导等),给结论先行的简析;不堆砌,只讲有信息增量的。
5. **持仓提示**(若用户持仓可得):按 `portfolio-advisor`,只读取持仓,给当日需关注的持仓事件与情景预案要点;**绝不下单**。无持仓数据则跳过或给"如需持仓分析请安装官方长桥技能(见 https://open.longbridge.com/skill/install.md)并完成 OAuth 授权,或粘贴持仓"。
6. **独立复核**:落盘前快速自检——数字有无来源/as-of?有无把推测当事实?有无变成买卖指令(应改为客观分析)?
7. **中文表达体检**:按 `stop-ai-slop-zh` 清理套话、排比三件套、名词化、抽象主语和金句收尾。只改表达,不得删来源、as-of 时间、关键数字、风险声明和事实/传闻/推断/推测分层。
8. **写文件并导出 PDF**:见下「输出」。

## 输出(必做)

写同名 Markdown + PDF 两个文件到本仓库:

```text
output/reports/美股研报_YYYY-MM-DD.md
output/reports/美股研报_YYYY-MM-DD.pdf
```

如用户另行指定外部「美股投研」目录,先做最小写入探测;不可写时仍必须回落到本仓库 `output/reports/`。不要把研报、图表或券商导出文件提交到 Git。

Markdown 写好后,调用日报专用 PDF 导出器:

```bash
python .agents/skills/daily-report/scripts/export_report.py --input output/reports/美股研报_YYYY-MM-DD.md --out output/reports/美股研报_YYYY-MM-DD.pdf --append-glossary --update-md --engine auto
```

`--engine auto` 优先使用 ReportLab 做真实段落排版;不可用时自动降级到 matplotlib 深色模板。导出后至少验证 Markdown 存在且标题可读、PDF 存在且大小大于 0、脚本输出包含 `profile=daily-report`。如 PDF 依赖或字体不可用,必须说明缺口,不能假装已导出。

文件结构:

```markdown
# 美股每日研报｜YYYY-MM-DD（美西/美东/北京）

## 市场温度
**定调**:<risk-on / risk-off / 震荡分歧>。最大矛盾:<一句话>。今日盯:<1-3 个关键触发>。

## 隔夜/收盘复盘
- 三大指数、Mag7、关键板块、涨跌主因
- 美债、美元、油金铜、BTC、VIX 读数与 as-of

## 宏观与盘前要闻
每条按 `事实 -> 已定价程度 -> 边际含义` 写,并标注 已证实 / 管理层声称 / 传闻 / 推断 / 推测。

## 事件日历
- 当日:数据、财报、讲话 + 共识与阈值 + 方向情景(美西+美东+北京)
- 未来 1-2 周:FOMC / CPI / PCE / NFP / 四巫日 / 国债拍卖 / 重点财报

## 社媒情绪
### WSB 股票雷达
| 排名 | 标的 | WSB 关注证据 | 喊单方向 | 常见玩法 | HYPE 阶段 | 风险/反向信号 |
|---|---|---|---|---|---|---|

### WSB 重点解读
每个标的必须写:核心叙事、散户在喊什么、支持/反对证据、是否纯 Meme、催化是否已证实、持续监控评分。

### HYPE 阶段热力图
- 早期/中期/末期/退潮分别有哪些标的;为什么。

### X/Twitter 交叉验证
- X 是否放大同一批 ticker/cashtag;是否出现 KOL/新闻驱动扩散;与 WSB 是否背离。

## 科技股/AI 供应链看点
- 卡点 thesis / 催化 / 证伪点；标信念档与证据成色

## 持仓提示
- 今日需盯的持仓事件 + 情景预案要点;只读,不下单

## 风险与证伪
- 什么会推翻今天的定调

## 主要来源
- 列出长桥、官方宏观日历、公司 IR/SEC、新闻源、WSB/X 抓取窗口和 as-of。

---
⚠️ 本研报为客观分析与信息整理，非投资建议。数据可能有误，决策请自行核实（DYOR）。
```

落盘后,验证 MD/PDF 都存在且标题可读;若运行环境支持文件呈现工具,把两个文件路径呈现给用户。正文只给一两句当日要点摘要。

## 约束
- ✅ 简体中文;结论先行;数字标来源 + as-of;区分 事实/传闻/推测。
- ✅ 落盘前用 `stop-ai-slop-zh` 做中文体检:句子要直,段落不要八股,但事实、数据、来源和证伪条件不能被润色掉。
- ✅ 社媒段必须以 WSB 为主,至少给出 5 个候选标的(数据不足时写明不足原因),并保留 HYPE 阶段、喊单方向、反向信号。
- ❌ 不给买卖指令、不预测精确点位、不下单。
- 控制长度:重信息增量,不为凑字数堆砌。
