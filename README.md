# Cucina

Cucina 是一个面向个人美股投研的中文工作台。它的核心目标不是自动交易,而是把持仓、宏观事件、市场情绪、个股基本面与技术面信号放到同一个分析框架里,生成可复核、可追踪、可执行但不代替用户决策的投资研究输出。

项目名 `cucina` 取“厨房”的意思:原始行情、持仓、新闻、期权链、社媒情绪和供应链信息都是食材;本项目负责清洗、分层、交叉验证,最后做成每日研报、个股分析和持仓预案。

## 核心功能

### 1. 使用长桥 MCP 分析持仓

通过长桥 Longbridge MCP 或官方长桥技能读取账户与持仓的只读数据,包括持仓列表、账户资产、实时行情、成本、浮盈亏、资金流、期权链、新闻、财报和机构预期等。

持仓分析的交付重点:

- 持仓结构:单票、板块、主题、因子和币种暴露。
- 风险集中度:是否押注同一条 AI 供应链、同一利率方向、同一高 beta 风格。
- 盈亏结构:浮盈浮亏分布、是否存在该止损未止损或该止盈未止盈的仓位。
- 情景预案:不同宏观、财报、政策、技术位触发时,如何加、减、持有、对冲或等待。
- 只读边界:不下单、不撤单、不改单、不移动资金、不创建定投、不创建价格提醒。

### 2. 定时生成每日美股研报

默认按用户本机美西时区 `America/Los_Angeles` 在美股常规交易日推送两次《每日美股研报》:美西 04:30(常规开盘美西 06:30 / 美东 09:30 前 2 小时,等价于美东 07:30)和美西 15:00(常规收盘美西 13:00 / 美东 16:00 后 2 小时,等价于美东 18:00)。夏令时/冬令时由美西时区自动换算,不固定写死北京时间。

盘前版覆盖隔夜复盘、当日盘前展望和未来 1-2 周重要事件;盘后版覆盖当日收盘复盘、盘后新闻/财报/指引、盘后异动和次日待验证事项。

研报通常整合以下模块:

- 宏观与事件:美联储、CPI/PCE、NFP、国债拍卖、财政债务、贸易政策、地缘风险。
- 市场内部结构:VIX、美债收益率、美元、油金铜、BTC、主要指数和关键板块。
- 社媒情绪:WSB 与 X/Twitter 热点、散户押注方向、Hype 阶段和拥挤度。
- 科技股与 AI 供应链:围绕供应链卡点、产能、客户、订单和证伪点提炼 1-3 条有信息增量的看点。
- 持仓提示:结合用户真实持仓,列出当天需要盯的事件、价位和情景触发条件。

研报输出必须使用简体中文,结论先行,数字标注来源和 as-of 时间,并明确区分事实、管理层声称、推断和推测。

正文落盘前使用 `stop-ai-slop-zh` 做一次中文表达体检:删套话、拆排比、去名词化,但不改事实、来源、as-of 时间和风险声明。

研报文件统一写入 `output/reports/` 子目录,推荐命名:

```text
output/reports/美股研报_YYYY-MM-DD.md
```

### 3. 个股技术分析

个股技术分析面向单只美股,优先从长桥读取只读行情、K 线、盘口、期权链和资金流数据,生成六维技术面体检。

六维结构:

- 格局:多头、空头、震荡或转折,参考 MA20/50/200、趋势斜率和高低点结构。
- 动量:RSI、MACD、ROC、量能配合和背离状态。
- 信号:突破、跌破、金叉、死叉、布林带、跳空缺口等触发情况。
- 趋势:ADX、DI、线性回归斜率、趋势通道,并输出趋势证据图。
- 期权结构:GEX、Call Wall、Put Wall、Gamma Flip、GEX PCR,并输出 GEX 柱形图。
- 资金与筹码:资金流、成交量分布、筹码密集区、压力位和支撑位,并输出资金/筹码图。

技术面分析只说明概率、结构和关键位,不承诺“必涨/必跌”,不替用户执行交易。

报告正文落盘前使用 `stop-ai-slop-zh` 清理 AI 味,让结论更像投资者写给自己看的复盘,同时保留所有指标、图表引用、术语解释和风险边界。

个股分析统一写入 `output/stock-analysis/<SYMBOL>/` 子目录。正文、趋势图、GEX 图、资金/筹码图和原始输入数据应放在同一个标的目录下,推荐命名:

```text
output/stock-analysis/AAOI/2026-06-09_AAOI_technical-analysis.md
output/stock-analysis/AAOI/AAOI_trend_real.png
output/stock-analysis/AAOI/AAOI_gex_est.png
output/stock-analysis/AAOI/AAOI_flow_real.png
```

### 4. 投资建议与交易预案

本项目可以给出投资建议,但建议必须以“分析 + 情景预案 + 触发条件”的形式呈现,不能写成无条件交易指令。

推荐输出方式:

```text
【代码】当前权重 X% | 浮盈亏 Y%
- thesis 状态:成立 / 动摇 / 破裂
- 基准看法:加 / 持 / 减 / 对冲 / 观察
- 情景预案:
  - 若 A 发生:考虑怎样处理,触发条件是什么
  - 若 B 发生:考虑怎样处理,风险是什么
  - 若 C 发生:什么信号会证伪原判断
- 关键位/日期:支撑、压力、财报、宏观数据、政策窗口
```

每次涉及加仓、减仓、止损、止盈、对冲或仓位调整,都必须说明:这是研究分析和预案,不是持牌投顾建议;最终交易由用户自行判断和执行。

### 5. MiroFish 情景仿真旁路

本项目可以把公开材料、已验证研究摘要和脱敏组合暴露整理成 MiroFish seed 包,供独立运行的 MiroFish 做多智能体情景推演。MiroFish 的输出只作为模拟推断或推测,必须回到 Cucina 中与长桥、官方公告、SEC/IR、宏观数据、新闻、技术面和期权结构交叉验证。

MiroFish 集成采用文件桥接方式,不把 MiroFish 源码复制进本仓库,不让 MiroFish 读取券商账户,也不把精确账户资产、成本、持仓数量、订单、流水或密钥默认写入 seed。标准工作流见 `docs/mirofish-integration.md`。

## 本项目已有模块

本仓库已经内置一组本地技能,分别放在 `.agents/skills/` 和 `.claude/skills/` 下:

- `us-equity-research`: 美股投研主控编排,负责在各子模块之间路由。
- `portfolio-advisor`: 长桥/券商只读持仓分析与情景预案。
- `daily-report`: 每日美股研报生成。
- `technical-analysis`: 单股技术面六维分析和图表输出。
- `tech-research`: 科技股与 AI 供应链卡点研究。
- `macro-events`: 宏观、地缘、财政、货币政策和事件日历分析。
- `wsb-sentiment`: WSB 与 X/Twitter 散户情绪分析。
- `mirofish-simulation`: MiroFish seed 包生成、情景仿真导入和模拟推断整理。
- `stop-ai-slop-zh`: 中文写作去 AI 味,用于每日研报、个股分析和持仓预案的落盘前表达体检。

技术分析图表的标准输出位置是 `output/stock-analysis/<SYMBOL>/`;不要把新生成的图表继续放在项目根目录。

## 输出目录

所有生成物统一放在 `output/` 下,并按类型进入子目录:

- `output/reports/`: 每日美股研报。
- `output/stock-analysis/<SYMBOL>/`: 个股分析正文、图表和中间数据。
- `output/portfolio/`: 后续用于持仓体检快照和组合层面预案。
- `output/simulations/`: MiroFish 情景仿真 seed、报告和摘要。
- `output/tmp/`: 临时抓取、清洗或图表输入数据。

`output/` 是本地生成目录,已加入 `.gitignore`,不要把里面的研报、图表、持仓快照或临时数据提交到版本库。

## 工作原则

- 简体中文优先:输出面向懂美股、懂基本金融术语的中文用户。
- 中文表达要自然:研报和个股分析写完后用 `stop-ai-slop-zh` 清理八股句式,但不能牺牲数据、来源和证伪条件。
- 数据优先:行情、持仓、新闻和财报都属于时效敏感信息,必须用实时或最新来源验证。
- 证据分层:事实、管理层声称、模型推断、市场传闻、个人判断必须分开写。
- 不编造数字:无法取到数据时明确说明缺口,可以降级为近似或请用户提供数据。
- 不自动交易:本项目是投研助手,不是交易执行系统。
- 不迎合多空倾向:框架不支持就说不支持,风险比机会大就直接说明。

## 典型使用方式

```text
分析我的持仓
出一份今天的美股研报
看一下 AAOI 的技术面
综合分析一下 NVDA,基本面和技术面都要
现在我的仓位该怎么调,给我几个情景预案
用 MiroFish 做一个 NVDA 财报情景推演
```
